## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> API (interrupt) | Interrupt sends user-composed system message to Hub via existing send endpoint |
| event metadata -> drawer | Cost and sentinel data read from event metadata — untrusted agent-supplied values displayed in UI |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-03 | Tampering | cost display | accept | Cost values come from event metadata set by Hub pipeline; client displays only, no financial decisions made from these values |
| T-06-04 | Spoofing | interrupt message | mitigate | Interrupt uses existing authenticated /messages endpoint which requires tenant session; no new auth surface |
| T-06-05 | Information Disclosure | sentinel flags in drawer | accept | Sentinel data is visible to all users who can access the run (no role-gating per D-12 deferral); operator-only gating deferred to Phase 9 |
| T-06-06 | Denial of Service | interrupt spam | mitigate | InterruptButton disables during isPending state; rate limiting inherited from existing send endpoint |
</threat_model>
