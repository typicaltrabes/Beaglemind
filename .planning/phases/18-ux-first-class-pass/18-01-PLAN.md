# 18-01 — MinIO public URL: fix broken image attachments

**Severity:** 🔴 CRITICAL (C1)
**Est:** 30 min

<objective>
User-facing image previews in run transcripts are silently broken because MinIO presigned URLs hardcode the internal Docker hostname (`http://beagle-console-minio-1:9000/...`), which:
1. Browser cannot resolve the hostname (private Docker DNS)
2. Plain HTTP triggers mixed-content block on the HTTPS console

**Root-cause file:** `apps/web/lib/storage/*` or wherever S3 client is built. Need to verify endpoint config + presigner.

**Output:** Image chips render their thumbnails; PDF "View" opens the PDF; MD "Download" downloads cleanly. All three from the run page.
</objective>

<tasks>

<task type="auto">
  <name>Task 1: Diagnose how S3 client is configured</name>
  <action>
1. Find the S3 client factory: grep `apps/web/lib` for `S3Client`, `getSignedUrl`, `endpoint`
2. Identify where the presigned URL is generated for the artifact-download endpoint
3. Check current `MINIO_ENDPOINT` / `MINIO_*` env vars on prod (`/opt/beagle-console/.env`)
4. Identify which subdomain MinIO should be reachable on publicly. Options:
   - Add Caddy reverse proxy at `minio.beaglemind.ai` → `beagle-console-minio-1:9000`
   - Or proxy through the web app's `/api/storage/[key]/route.ts` (server reads from MinIO with internal client, streams to browser)
   - Pick option B if it's simpler — avoids Caddy config changes + keeps auth at the web layer.
  </action>
  <done>Decision recorded in this plan's notes: chosen approach + file list.</done>
</task>

<task type="auto">
  <name>Task 2: Implement chosen approach</name>
  <action>
**If Option B (proxy through web app):**
1. The `/api/artifacts/[id]/download/route.ts` currently issues a 307 redirect to MinIO. Change it to:
   - Validate auth + tenant scope
   - Server-side fetch from MinIO using internal endpoint
   - Stream bytes back to browser with correct content-type + content-disposition
2. Image previews can now point directly at `/api/artifacts/${id}/download`
3. PDFs in the View button continue to use the same path
4. No client-side changes needed

**If Option A (Caddy + public URL):**
1. Update `/opt/sonic-hq/caddy/Caddyfile` to add `minio.beaglemind.ai` reverse_proxy block
