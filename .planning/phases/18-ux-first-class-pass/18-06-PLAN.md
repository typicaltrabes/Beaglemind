# 18-06 — Composer + per-message interactions

**Severity:** 🟡 MEDIUM (M4+M6+M7)
**Est:** 30 min

<objective>
Conversation surface lacks standard chat-app affordances:
- Can't copy a single message
- Can't reply-to a specific agent message
- Can't regenerate a failed agent reply
- Composer placeholder implies Mo is the lead ("Describe what you want Mo to research…")
- Dashboard composer has no paperclip — image attachments require start-then-follow-up dance

**Output:** Hover any message → copy/reply/regenerate menu. New runs can start with attachments.
</objective>

<tasks>

<task type="auto">
  <name>Task 1: Per-message hover actions</name>
  <action>
1. Add hover-overlay action menu (3 small icon buttons) on each message bubble:
   - Copy text (clipboard)
   - Reply (inserts `@AgentName ` prefix into composer + scrolls composer into view)
   - Regenerate (only for failure-bubbles): re-call the agent, replace the failure bubble with the new response
2. Position: top-right of bubble, only visible on hover
3. Keyboard accessible (focus-within reveals)
  </action>
  <done>Hover any message → 2-3 action icons appear; clicking copies/replies/regenerates.</done>
</task>

<task type="auto">
  <name>Task 2: Composer placeholder</name>
  <action>
1. Dashboard composer: "Describe what you want Mo to research…" → "Ask the team a question…"
2. Run-page composer: "Send a message…" stays (works fine in conversation context)
3. Optional: rotate placeholder examples on focus ("…or upload a chart and ask for analysis")
  </action>
  <done>Placeholder doesn't single-out Mo.</done>
</task>

<task type="auto">
  <name>Task 3: Paperclip on dashboard composer</name>
  <action>
1. Dashboard composer currently has only textbox + Start Sprint button
2. Add paperclip + drag-drop matching the run-page composer's component
3. When dashboard composer has attachments, the run is started WITH the attachments included on the first user message
4. Backend route already accepts `imageAttachments` per Phase 17.1 schema — wire the dashboard call to use the same endpoint
  </action>
  <done>Customer can start a sprint with images directly, no two-step dance.</done>
</task>

</tasks>

<success_criteria>
- Hover actions appear on all message bubbles
- Copy/Reply work
- Regenerate works on failure bubbles
- Dashboard composer accepts attachments
- Composer placeholder is team-oriented
</success_criteria>
