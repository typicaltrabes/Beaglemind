    Per D-13: Generated on-demand. Queries events since last_active_at.

    1. **apps/web/app/api/digest/route.ts** -- GET endpoint:
       - Get authenticated user and tenant context (same pattern as other API routes)
       - Read user's `lastActiveAt` from users table. If null, default to 24 hours ago.
       - Query tenant schema for activity since lastActiveAt:
         a. **Runs progressed**: SELECT from runs WHERE updatedAt > lastActiveAt. Return id, projectId, status, prompt (truncated).
         b. **Artifacts delivered**: SELECT from artifacts WHERE createdAt > lastActiveAt. Return id, runId, filename, sizeBytes.
         c. **Questions answered**: SELECT from questions WHERE answeredAt > lastActiveAt. Return id, runId, agentId, content (truncated), answer (truncated).
         d. **Questions pending**: SELECT from questions WHERE answer IS NULL. Return count.
       - Update user's lastActiveAt to NOW() after building response
       - Return JSON: `{ since: lastActiveAt, runs: [...], artifacts: [...], answeredQuestions: [...], pendingCount: number }`
       - Guard: If no activity, return empty arrays with `since` timestamp.

    2. **apps/web/lib/hooks/use-digest.ts**:
       - TanStack Query hook: `useDigest()` that fetches GET /api/digest
       - staleTime: 5 minutes (digest doesn't change frequently)
       - Return typed `DigestData` object

       - Uses `useDigest()` hook
       - Renders sections:
