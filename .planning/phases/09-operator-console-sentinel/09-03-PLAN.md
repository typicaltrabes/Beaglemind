    1. **POST /api/operator/break-glass** (per D-09):
       - Gate: requireOperator().
       - Body: `{ tenantId: string, reason: string }`.
       - Validate: tenantId exists in shared.tenants. Reason is non-empty, min 10 chars.
       - Check: no active (non-expired, non-revoked) break-glass session already exists for this operator + tenant.
       - Create: insert into tenant's breakGlassAudit table:
         ```
         operatorId: session.userId,
         operatorEmail: session.user.email,
         reason,
         grantedAt: new Date(),
         expiresAt: new Date(Date.now() + 4 * 60 * 60 * 1000), // 4 hours per D-09
         ```
       - Return: `{ id, grantedAt, expiresAt }` with status 201.

    2. **GET /api/operator/break-glass** (list active sessions):
       - Gate: requireOperator().
       - Query all tenants, for each check breakGlassAudit where operatorId = current user AND expiresAt > now AND revokedAt IS NULL.
       - Return: `[{ tenantId, tenantName, reason, grantedAt, expiresAt }]`.

    3. **GET /api/operator/break-glass/[tenantId]/runs** (per D-11):
       - Gate: requireOperator().
       - Validate active break-glass session: query tenant's breakGlassAudit where operatorId = current user, expiresAt > now, revokedAt IS NULL. If none, return 403 `{ error: 'No active break-glass session' }`.
       - Query tenant's runs joined with projects. Return array of runs with project name, status, prompt, createdAt.

    4. **GET /api/operator/break-glass/[tenantId]/runs/[runId]** (per D-11):
       - Gate: requireOperator() + active break-glass session check (same as above).
       - Query tenant's events for this runId, ordered by sequenceNumber.
       - Return: `{ run: { id, status, prompt, createdAt }, events: HubEventEnvelope[] }`.
       - This gives operator read-only view of full conversation content during break-glass window.

    acceptance_criteria:

    - GET lists active sessions
    - Tenant data endpoints verify active break-glass before returning data
    - Expired sessions return 403

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Operator -> tenant data | Break-glass bypasses tenant isolation; must be time-boxed and audited |
| Tenant admin -> audit log | Admin can see operator access records for their org |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-07 | Elevation of Privilege | break-glass endpoints | mitigate | requireOperator() + active session check on every data access. 4hr hard expiry enforced server-side (D-09). |
| T-09-08 | Repudiation | break-glass audit | mitigate | Every access creates immutable audit record in tenant schema with operatorId, reason, timestamps (D-10). Tenant admin sees all records. |
| T-09-09 | Information Disclosure | /api/operator/break-glass/[tenantId]/runs/[runId] | mitigate | Returns events only during active break-glass window. Expired = 403. No caching headers. |
| T-09-10 | Tampering | breakGlassAudit records | mitigate | Audit records stored in tenant schema (operator cannot modify tenant DB directly). expiresAt set at creation, not modifiable via API. |
</threat_model>
