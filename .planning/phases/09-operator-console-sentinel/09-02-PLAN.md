    read_first:
    - apps/web/lib/operator.ts (requireOperator from Plan 01)
    - apps/web/app/api/runs/route.ts (pattern for existing API routes)
    - packages/db/src/schema/shared.ts (tenants table)
    - packages/db/src/schema/tenant.ts (createTenantSchema)
    - packages/db/src/client.ts (db instance)

    All operator API routes must call `requireOperator()` first. If not operator, return 403.

    1. **GET /api/operator/health** (per D-04):
       - Check Postgres: run `SELECT 1` via db, catch errors.
       - Check Redis: try connecting to `process.env.REDIS_URL` with ioredis (dynamic import like SSE route does), ping.
       - Check MinIO: import getMinioClient from @beagle-console/db, call `client.listBuckets()`, catch errors.
       - Check Hub: fetch `${process.env.AGENT_HUB_URL ?? 'http://localhost:4100'}/health`, catch errors.
       - Return: `{ postgres: { status: 'ok'|'error', latencyMs }, redis: { status, latencyMs }, minio: { status, latencyMs }, hub: { status, latencyMs, agents? } }`

    2. **GET /api/operator/stats** (per D-04):
       - Query `shared.tenants` for count.
       - For each tenant, query runs table where status IN ('executing', 'approved', 'planned') for active runs count.
       - For cost: query events across tenants for type='cost_update' in last 24h/7d/30d (or return 0 if no cost events yet -- this is fine for Phase A).
       - Return: `{ tenantCount, activeRunsCount, cost: { last24h, last7d, last30d } }`

    3. **GET /api/operator/runs** (per D-05):
       - Query all tenants from shared.tenants.
       - For each tenant, query runs with status NOT IN ('completed', 'cancelled') joined with projects for project name.
       - Return array: `[{ tenantName, tenantSlug, projectName, runId, status, createdAt, prompt }]`
       - Sort by createdAt desc. Limit to 100.

    4. **GET /api/operator/sentinel** (per D-08):
       - Query all tenants.
       - For each tenant, query events where type='sentinel_flag' OR (type='system' AND content->>'source'='sentinel').
       - Return array: `[{ tenantName, runId, text, severity, timestamp, agentId }]`
       - Sort by timestamp desc. Limit to 200.

    Pattern note: Use `sql` template literals from drizzle-orm for cross-schema queries. For each tenant, construct the schema dynamically via createTenantSchema(tenantId).

    acceptance_criteria:
    - All 4 API routes exist and gate on requireOperator()
    - Health route checks all 4 services
    - Stats/runs/sentinel routes query across tenants

| Browser -> /api/operator/* | Untrusted client requests to operator endpoints |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-04 | Elevation of Privilege | /api/operator/* routes | mitigate | Every route calls requireOperator() which checks isOperator flag in DB. Non-operators get 403. |
| T-09-05 | Information Disclosure | /api/operator/runs | mitigate | Only returns run metadata (status, prompt snippet), not full conversation content. Content access requires break-glass (Plan 03). |
| T-09-06 | Denial of Service | Cross-tenant queries | accept | Queries iterate tenants sequentially; with Year-1 scale (<10 tenants) this is fine. Add pagination if needed later. |
</threat_model>
