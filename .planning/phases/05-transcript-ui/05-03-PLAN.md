**2. Rewrite message-list.tsx (per D-12, D-13, D-14, D-02, D-05, D-09, D-18):**

This is a FULL REWRITE of the existing message-list.tsx. The new version:

**Data model — build a flat render list from scenes:**

Create a `useMemo` that transforms `scenes` + `events` + `eventOrder` into a flat array of render items:

```typescript
type RenderItem =
  | { kind: 'scene-divider'; sceneId: string; sceneName: string }
  | { kind: 'event'; seq: number }
  | { kind: 'collapse-fold'; range: CollapsibleRange };
```

Algorithm:
1. For each scene in `scenes`:
   a. If scene.name is non-empty, push a `scene-divider` item
   b. Run `detectCollapsibleRanges(scene.eventSequences, events)` to find collapsible sections within this scene
   c. Walk scene.eventSequences. For each sequence:
      - If this seq is the start of a CollapsibleRange, push a `collapse-fold` item and skip all seqs in that range
      - Otherwise push an `event` item
2. This produces a flat list suitable for Virtuoso

**Virtuoso setup (per D-12, D-13, D-14):**

```tsx
import { Virtuoso, type VirtuosoHandle } from 'react-virtuoso';
```

Use `<Virtuoso>` with:
- `data={renderItems}` — the flat render list
- `itemContent={(index, item) => renderItem(item)}` — dispatch by `item.kind`
- `followOutput="smooth"` — auto-scroll to bottom when new items added (per D-14)
- `atBottomStateChange={(atBottom) => setIsAtBottom(atBottom)}` — track if user scrolled up
- When `isAtBottom` is false and user has scrolled up, `followOutput` should be disabled. Use a ref: `followOutput={isAtBottom ? "smooth" : false}`
- `overscan={200}` — render 200px of extra items above/below viewport for smooth scrolling
- `className="h-full"` with the parent being a flex-1 container

**Render dispatch:**

```typescript
function renderItem(item: RenderItem): React.ReactNode {
  switch (item.kind) {
    case 'scene-divider':
      return <SceneDivider name={item.sceneName} />;
    case 'collapse-fold':
      return (
        <CollapseFold
          range={item.range}
          events={events}
          renderEvent={(seq) => renderSingleEvent(seq)}
        />
      );
    case 'event':
      return renderSingleEvent(item.seq);
  }
}
```

The `renderSingleEvent` function handles the event type dispatch (same logic as old `renderEvent` but using the new `AgentMessage` component from Plan 01 instead of the old inline one):

```typescript
function renderSingleEvent(seq: number): React.ReactNode {
  const event = events[seq];
  if (!event) return null;
  switch (event.type) {
    case 'plan_proposal':
      return <PlanCard event={event} runId={runId} />;
    case 'question':
      return <QuestionCard event={event} runId={runId} />;
    case 'artifact':
      return <ArtifactCard event={event} />;
    case 'agent_message':
      return <AgentMessage event={event} />;
    case 'state_transition':
      return <StateTransitionMessage event={event} />;
    default:
      return null;
  }
}
```

Keep the `StateTransitionMessage` inline component from the old file (centered dim text showing from/to).
