The attachment block format from CONTEXT.md (verbatim — DO NOT alter):
```
--- USER ATTACHMENTS ---
[1] filename.pdf (PDF, 142 KB)
<extracted text or "(image — see attached)">
[2] screenshot.png (PNG, 38 KB)

Mapping rules (per CONTEXT specifics + PATTERNS recommendation):
- `(MIME_LABEL, formatted_size)` — derive a human label from mimeType (PDF, DOCX, PNG, JPEG, WEBP, TXT, MD)
- If `extractedText` is non-null: emit it on the next line (raw, no markdown fencing)
- If `extractedText` is null AND mimeType starts with `image/`: emit `(image — included with this message)` (NOT "see attached" — CONTEXT specifies "included with this message")
- If `extractedText` is null AND mimeType is NOT an image: emit `(no extracted text available)`

Trailing whitespace: the block ends with `--- END ATTACHMENTS ---` followed by exactly two newlines (`\n\n`) before the user content begins.

const MIME_LABELS: Record<string, string> = {
  'application/pdf': 'PDF',
  'application/vnd.openxmlformats-officedocument.wordprocessingml.document': 'DOCX',
  'image/png': 'PNG',
  'image/jpeg': 'JPEG',
  'image/webp': 'WEBP',
  'text/plain': 'TXT',
  'text/markdown': 'MD',
};

function mimeLabel(mimeType: string): string {
  return MIME_LABELS[mimeType] ?? mimeType;
}

interface ArtifactRow {
  filename: string;
  mimeType: string;
  sizeBytes: number;
  extractedText: string | null;
}

function buildAttachmentBlock(rows: ArtifactRow[]): string {
  if (rows.length === 0) return '';
  const lines: string[] = ['--- USER ATTACHMENTS ---'];
  rows.forEach((r, i) => {
    lines.push(`[${i + 1}] ${r.filename} (${mimeLabel(r.mimeType)}, ${formatSize(r.sizeBytes)})`);
    if (r.extractedText) {
      lines.push(r.extractedText);
    } else if (r.mimeType.startsWith('image/')) {
      lines.push('(image — included with this message)');
    } else {
      lines.push('(no extracted text available)');
    }
  });
  lines.push('--- END ATTACHMENTS ---');
  return lines.join('\n') + '\n\n';
}
```

D) Rewrite the POST handler body (replacing lines 47-79). Keep the existing run-status update + hubClient.startRun + return shape unchanged; insert the attachment lookup + prepend in between:
```ts
export async function POST(
  request: Request,
  { params }: { params: Promise<{ id: string }> },
) {
  try {
    const { id: runId } = await params;
    const { tenantId } = await requireTenantContext();
    const { db: tdb, schema } = getTenantDb(tenantId);

    const body = await request.json();
    const { content, attachmentIds } = SendMessageBody.parse(body);

    let attachmentBlock = '';
    if (attachmentIds.length > 0) {
      const rows = await tdb
        .select({
          id: schema.artifacts.id,
          filename: schema.artifacts.filename,
          mimeType: schema.artifacts.mimeType,
          sizeBytes: schema.artifacts.sizeBytes,
          extractedText: schema.artifacts.extractedText,
        })
        .from(schema.artifacts)
        .where(
          and(
            inArray(schema.artifacts.id, attachmentIds),
            eq(schema.artifacts.runId, runId),
            eq(schema.artifacts.agentId, 'user'),
          ),
        );

      // Strict ownership check — every requested id must resolve. If any
      // attachmentId was deleted, belongs to a different run, or was produced
      // by an agent (not user), reject with 404.
      if (rows.length !== attachmentIds.length) {
        return NextResponse.json(
          { error: 'attachment not found' },
          { status: 404 },
        );
      }

      // Preserve client-specified order (db .where(inArray(...)) returns rows
      // in arbitrary order; reorder to match attachmentIds)
      const byId = new Map(rows.map((r) => [r.id, r]));
      const ordered = attachmentIds
        .map((id) => byId.get(id))
        .filter((r): r is NonNullable<typeof r> => Boolean(r));
      attachmentBlock = buildAttachmentBlock(ordered);
    }

    // Update run to executing if it was completed (continue conversation)
    await tdb
      .update(schema.runs)
      .set({ status: 'executing', updatedAt: new Date() })
      .where(eq(schema.runs.id, runId));

    // Hub is the sole writer of events — it persists the user message through
    // its SequenceCounter and then kicks off the round-table. Propagate failure
    // to the client so it can retry rather than silently losing the message.
    await hubClient.startRun({
      runId,
      tenantId,
      prompt: attachmentBlock + content,
    });

    return NextResponse.json({ ok: true });
  } catch (error) {
    if (error instanceof z.ZodError) {
      return NextResponse.json({ error: error.message }, { status: 400 });
    }
    console.error('POST /api/runs/[id]/messages error:', error);
    return NextResponse.json({ error: 'Internal server error' }, { status: 500 });
  }
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → POST /api/runs/[id]/messages | attachmentIds is user-controlled. Server MUST verify each ID belongs to this tenant + run + agentId='user'. |
| route → hub | The combined `attachmentBlock + content` string flows to hubClient.startRun. The hub treats it as plaintext; LLM prompt-injection from extracted_text is a known concern. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-03-01 | Spoofing — caller references another tenant's artifactId | tdb scope | mitigate | `getTenantDb(tenantId)` selects from the caller's tenant schema. Cross-tenant IDs would not even resolve to a row. |
| T-17-03-02 | Tampering — caller references another run's artifact (cross-run reuse) | WHERE clause | mitigate | The SELECT has `eq(schema.artifacts.runId, runId)` ensuring the artifact was uploaded to this exact run. CONTEXT D-08 explicitly disallows cross-run file libraries. |
| T-17-03-03 | Tampering — caller references an agent-produced artifact (e.g. impersonating Mo's output as user input) | WHERE clause | mitigate | The SELECT has `eq(schema.artifacts.agentId, 'user')` — only user-uploaded files can be referenced. |
| T-17-03-04 | LLM prompt injection via extracted text | extracted_text → prompt | accept | The user controls the file they uploaded; same trust level as the user's prompt itself. The structured `--- USER ATTACHMENTS ---` block makes the boundary explicit to the agent (prompt engineering); we do NOT escape the body because that would defeat its purpose. Standard LLM-prompt-injection acceptance — same model the team accepts for all other user-controlled prompt content. |
| T-17-03-05 | DoS — caller sends max-size 4 attachments × 50K chars each = 200K chars on every message | total prompt size | accept | 200K chars is well within Anthropic Opus / OpenClaw's context window (~200K tokens). Hub already enforces no separate per-message rate limit; the existing user message rate limits cover this. Re-evaluate if observed prompt sizes break model context windows — would log to Phase 18 backlog. |
| T-17-03-06 | Information disclosure — extracted text from a deleted/revoked artifact still flows to the agent | strict 404 path | mitigate | If an artifact is deleted between upload and message-send, the SELECT count mismatch triggers 404. The user-visible error is generic; no info leaks. |
</threat_model>
