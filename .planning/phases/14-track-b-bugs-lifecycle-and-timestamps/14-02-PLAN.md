<objective>
Bug 2 from CONTEXT.md `<decisions>`: every run is stuck on `executing` for days because the hub round-table function (`apps/agent-hub/src/http/routes.ts:runRoundTable`) iterates the agents, logs `Round-table discussion complete`, and exits — without ever writing `runs.status = 'completed'`. CONTEXT.md confirms this is verifiable by greppability: `apps/agent-hub/src` has zero references to `runs.status` today.

Fix: at the very end of `runRoundTable` (after the for-loop, before the existing `log.info({...}, 'Round-table discussion complete')`), write `runs.status = 'completed'` to the tenant's runs table and emit a corresponding `state_transition` event so live UIs flip the chip without a reload. Best-effort: if the DB write fails, log and return — don't throw, don't rollback the agent responses (they're already persisted by `persistAndPublish` inside the loop).

Output:
- `apps/agent-hub/src/http/routes.ts` imports `db` and `createTenantSchema` from `@beagle-console/db` and `eq` from `drizzle-orm`.
- `runRoundTable` ends with: `db.update(runs).set({ status: 'completed', updatedAt: ... }).where(eq(runs.id, runId))` followed by `router.persistAndPublish(tenantId, { type: 'state_transition', ... from: 'executing', to: 'completed' })`. Both wrapped in independent try/catches so a failure in either is logged and ignored.

Independent of 14-01 (different file, different subsystem). Runs in Wave 1 in parallel with the SSE replay fix.
