1. Create `apps/web/middleware.ts` per D-07, D-08, D-09:
   - CRITICAL: This runs in Edge Runtime. Do NOT import auth.ts or any database code. Only use `getSessionCookie` from "better-auth/cookies" for lightweight cookie check.
   - Import `getSessionCookie` from "better-auth/cookies"
   - Import `NextResponse` from "next/server" and `NextRequest` type
   - Define `publicPaths` array: ["/login", "/signup", "/mfa-challenge", "/mfa-setup", "/accept-invite", "/no-org"]
   - In the middleware function:
     a. Extract `pathname` from `request.nextUrl`
     b. If pathname starts with any publicPaths entry, return `NextResponse.next()`
     c. If pathname starts with "/api/auth", return `NextResponse.next()`
     d. Call `getSessionCookie(request)` -- if no cookie, redirect to `/login`
     e. Otherwise return `NextResponse.next()`
   - Export `config` with matcher: `["/((?!_next/static|_next/image|favicon.ico).*)"]`
   - NOTE: Full session validation + org check happens in `requireTenantContext()`, not in middleware. Middleware only does the lightweight cookie existence check (per research Pitfall 2: Edge Runtime cannot access database).

2. Create `apps/web/lib/get-tenant.ts` per D-08, D-09:
   - Import `auth` from "./auth"
   - Import `headers` from "next/headers"
   - Import `redirect` from "next/navigation"
   - Export async function `requireTenantContext()`:
     a. `const session = await auth.api.getSession({ headers: await headers() })`
     b. If no session, `redirect("/login")`
     c. `const tenantId = session.session.activeOrganizationId`
     d. If no tenantId, `redirect("/no-org")`
     e. Return `{ session, tenantId }`
   - This function is called from server components and API route handlers. It runs in Node.js runtime (not Edge), so it CAN access the database.
   - Also export a convenience function `getTenantDb(tenantId: string)` that:
     a. Imports `createTenantSchema` from "@beagle-console/db"
     b. Imports `db` from "@beagle-console/db"
     c. Returns `{ db, ...createTenantSchema(tenantId) }` -- giving the caller a scoped query builder
   - OR: simply re-export/wrap the existing pattern. The key is that every API route starts with `const { tenantId } = await requireTenantContext(); const tenantDb = getTenantDb(tenantId);`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> middleware | Every request checked for session cookie |
| middleware -> server component | Full session + org validated server-side |
| server component -> database | Tenant-scoped via getTenantDb(tenantId) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Spoofing | middleware.ts | mitigate | getSessionCookie checks for valid session token. Full validation in requireTenantContext() verifies against database. |
| T-02-05 | Information Disclosure | get-tenant.ts | mitigate | activeOrganizationId from session determines tenant scope. No tenant ID from client input -- always from authenticated session. |
| T-02-06 | Elevation of Privilege | (dashboard) routes | mitigate | requireTenantContext() in dashboard layout ensures every child route requires auth + org membership. No org = redirect to /no-org. |
| T-02-07 | Tampering | login form | mitigate | Better Auth handles credential validation. Client-side form uses controlled inputs, server validates via auth.api. |
</threat_model>
