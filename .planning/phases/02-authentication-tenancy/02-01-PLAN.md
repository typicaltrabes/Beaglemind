<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> /api/auth/* | Untrusted auth input (email, password, TOTP codes) |
| auth server -> database | Session/credential storage in shared schema |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | /api/auth/[...all] | mitigate | Better Auth handles CSRF via origin check, password hashing via bcrypt. Rate limiting deferred to Claude's discretion (can add later). |
| T-02-02 | Information Disclosure | auth-schema.ts | mitigate | Passwords stored as bcrypt hashes by Better Auth. TOTP secrets encrypted by Better Auth 2FA plugin. No plaintext credentials in schema. |
| T-02-03 | Tampering | session cookies | mitigate | Better Auth sets httpOnly, secure (when BETTER_AUTH_URL is https), sameSite cookies. nextCookies() plugin handles Next.js cookie integration. |
</threat_model>
