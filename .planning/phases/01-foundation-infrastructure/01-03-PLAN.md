Check if the user has a GitHub org and what its slug is. If unsure, use lowercase `beaglemind` as the GHCR org path (GitHub lowercases org names in GHCR URLs).

Create `.github/workflows/deploy.yml`:

```yaml
name: Deploy

on:
  push:
    branches: [main]

env:
  REGISTRY: ghcr.io
  ORG: beaglemind

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write

    steps:
      - uses: actions/checkout@v4

      - name: Log in to GHCR
        uses: docker/login-action@v3
        with:
          registry: ${{ env.REGISTRY }}
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Build and push console-web
        uses: docker/build-push-action@v6
        with:
          context: .
          file: apps/web/Dockerfile
          push: true
          tags: ${{ env.REGISTRY }}/${{ env.ORG }}/console-web:latest,${{ env.REGISTRY }}/${{ env.ORG }}/console-web:${{ github.sha }}
          cache-from: type=gha
          cache-to: type=gha,mode=max

      - name: Build and push console-agent-hub
        uses: docker/build-push-action@v6
        with:
          context: .
          file: apps/agent-hub/Dockerfile
          push: true
          tags: ${{ env.REGISTRY }}/${{ env.ORG }}/console-agent-hub:latest,${{ env.REGISTRY }}/${{ env.ORG }}/console-agent-hub:${{ github.sha }}
          cache-from: type=gha
          cache-to: type=gha,mode=max

      - name: Build and push console-worker
        uses: docker/build-push-action@v6
        with:
          context: .
          file: apps/worker/Dockerfile
          push: true
          tags: ${{ env.REGISTRY }}/${{ env.ORG }}/console-worker:latest,${{ env.REGISTRY }}/${{ env.ORG }}/console-worker:${{ github.sha }}
          cache-from: type=gha
          cache-to: type=gha,mode=max

      - name: Deploy to BeagleHQ
        uses: appleboy/ssh-action@v1
        with:
          host: 46.224.167.166
          username: lucas
          key: ${{ secrets.SSH_PRIVATE_KEY }}
          script: |
            echo ${{ secrets.GITHUB_TOKEN }} | docker login ghcr.io -u ${{ github.actor }} --password-stdin
            cd /opt/beagle-console
            docker compose pull
            docker compose up -d --remove-orphans
            docker image prune -f
```

**Key details:**
- Tags include both `latest` and SHA-pinned tag for rollback capability
- `cache-from: type=gha` uses GitHub Actions cache for faster rebuilds
- `docker image prune -f` on BeagleHQ cleans up old images to save disk
- `--remove-orphans` cleans up removed services
- `GITHUB_TOKEN` is auto-provided by GitHub Actions with `packages:write` scope
- `SSH_PRIVATE_KEY` must be added as a repository secret by the user

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions -> BeagleHQ | SSH key grants deployment access to production server |
| GitHub Actions -> GHCR | GITHUB_TOKEN grants package push to org registry |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-10 | Spoofing | SSH key in GitHub Secrets | mitigate | SSH key stored as encrypted GitHub secret. Key should have minimal permissions (only the lucas user, not root). |
| T-01-11 | Tampering | Deploy script | mitigate | Workflow only triggers on push to main (D-16: branch protection required). docker compose pull ensures signed images from GHCR. |
| T-01-12 | Information Disclosure | GITHUB_TOKEN in SSH script | accept | Token is ephemeral (expires after workflow). Used only to authenticate docker login on BeagleHQ. Low risk. |
</threat_model>
