**4. docker/docker-compose.yml** (per D-04: at /opt/beagle-console/, D-05: external networks, D-07: memory limits, D-08: MinIO):

```yaml
services:
  console-web:
    image: ghcr.io/beaglemind/console-web:latest
    build:
      context: ..
      dockerfile: apps/web/Dockerfile
    restart: unless-stopped
    networks:
      - beaglehq_backend
      - beaglehq_frontend
    environment:
      - DATABASE_URL=${DATABASE_URL}
      - REDIS_URL=${REDIS_URL}
      - MINIO_ENDPOINT=minio
      - MINIO_PORT=9000
      - MINIO_ACCESS_KEY=${MINIO_ACCESS_KEY}
      - MINIO_SECRET_KEY=${MINIO_SECRET_KEY}
    deploy:
      resources:
        limits:
          memory: 512M

  console-agent-hub:
    image: ghcr.io/beaglemind/console-agent-hub:latest
    build:
      context: ..
      dockerfile: apps/agent-hub/Dockerfile
    restart: unless-stopped
    networks:
      - beaglehq_backend
    environment:
      - DATABASE_URL=${DATABASE_URL}
      - REDIS_URL=${REDIS_URL}
    deploy:
      resources:
        limits:
          memory: 256M

  console-worker:
    image: ghcr.io/beaglemind/console-worker:latest
    build:
      context: ..
      dockerfile: apps/worker/Dockerfile
    restart: unless-stopped
    networks:
      - beaglehq_backend
    environment:
      - DATABASE_URL=${DATABASE_URL}
      - REDIS_URL=${REDIS_URL}
      - MINIO_ENDPOINT=minio
      - MINIO_PORT=9000
      - MINIO_ACCESS_KEY=${MINIO_ACCESS_KEY}
      - MINIO_SECRET_KEY=${MINIO_SECRET_KEY}
    deploy:
      resources:
        limits:
          memory: 256M

  minio:
    image: minio/minio:latest
    command: server /data --console-address ":9001"
    restart: unless-stopped
    networks:
      - beaglehq_backend
    volumes:
      - minio_data:/data
    environment:
      - MINIO_ROOT_USER=${MINIO_ACCESS_KEY}
      - MINIO_ROOT_PASSWORD=${MINIO_SECRET_KEY}
    deploy:
      resources:
        limits:
          memory: 256M

  minio-init:
    image: minio/mc:latest
    depends_on:
      minio:
        condition: service_started
    entrypoint: >
      /bin/sh -c "
      sleep 5;
      mc alias set local http://minio:9000 $${MINIO_ACCESS_KEY} $${MINIO_SECRET_KEY};
      mc mb local/system --ignore-existing;
      echo 'MinIO initialization complete';
      "
    networks:
      - beaglehq_backend
    restart: "no"

networks:
  beaglehq_backend:
    external: true
  beaglehq_frontend:
    external: true

volumes:
  minio_data:
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Docker host -> containers | Container escape risk, mitigated by non-root USER |
| External network -> Caddy | Public internet access to reverse proxy |
| Environment variables | Database credentials, MinIO keys in .env files |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | Elevation of Privilege | Dockerfiles | mitigate | All Dockerfiles use non-root USER (nextjs for web, node for agent-hub/worker). Memory limits prevent resource exhaustion. |
| T-01-06 | Information Disclosure | docker/.env.example | mitigate | .env.example contains placeholder values only. Actual .env on server never committed. .gitignore excludes .env*. |
| T-01-07 | Spoofing | Caddy TLS | mitigate | Caddy auto-provisions TLS via Let's Encrypt ACME. HTTPS enforced by default. |
| T-01-08 | Information Disclosure | MinIO | mitigate | MinIO credentials via environment variables, not hardcoded. MinIO only accessible on internal Docker network (beaglehq_backend), not exposed to host ports. |
| T-01-09 | Denial of Service | Docker memory limits | mitigate | Per D-07: web 512M, agent-hub 256M, worker 256M, minio 256M. Total ~1.3GB, leaving headroom on 8GB VPS. |
</threat_model>
