**1. Create `apps/agent-hub/src/handlers/message-router.ts`** per D-05, D-06:
- Function `mapOpenClawToEvent(agentId: string, msg: OpenClawInbound, runId: string, tenantId: string)` returns partial HubEventEnvelope (without sequenceNumber/timestamp — those are added by EventStore):
  - chat.response -> { type: 'agent_message', agentId, runId, tenantId, content: { text: msg.content }, metadata: { done: true } }
  - chat.stream -> { type: 'agent_message', agentId, runId, tenantId, content: { text: msg.content }, metadata: { done: msg.done, streaming: true } }
  - chat.typing -> { type: 'system', agentId, runId, tenantId, content: { event: 'typing' }, metadata: {} }
  - chat.error -> { type: 'system', agentId, runId, tenantId, content: { event: 'error', error: msg.error }, metadata: {} }

- Class `MessageRouter` that orchestrates the full pipeline:
  - Constructor takes: EventStore, RedisPublisher, logger
  - `async handleAgentMessage(agentId: string, rawMsg: unknown, runId: string, tenantId: string)`:
    1. Parse rawMsg with OpenClawInbound Zod schema. If invalid, log warning and return (drop invalid messages per T-03-01).
    2. Map to event via mapOpenClawToEvent
    3. Call eventStore.persist(tenantId, event) — this assigns sequenceNumber and timestamp (D-08: persist BEFORE broadcast)
    4. Call redisPublisher.publish(persistedEvent) — only after successful persistence
    5. Log the event at debug level

**2. Create `apps/agent-hub/src/http/routes.ts`** per D-14:
- Export route handler functions that take the Hub's components as dependencies:

- `handleSend(body, registry, router)`: Validates body (expects { agentId, content, runId, tenantId }). Constructs OpenClawOutbound message. Calls registry.send(agentId, outbound). Also persists a 'user' type event via router for the user's message. Returns { ok: true }.

- `handleRunStart(body, registry, router)`: Validates body (expects { runId, tenantId, prompt, targetAgent }). Defaults targetAgent to 'mo'. Constructs OpenClawOutbound with the prompt. Sends to target agent. Persists a state_transition event (type: 'state_transition', content: { from: 'pending', to: 'executing' }). Returns { ok: true, runId }.

- `handleRunStop(body, registry, router)`: Validates body (expects { runId, tenantId }). Persists a state_transition event (content: { from: 'executing', to: 'cancelled' }). Does NOT send to agent (agent-side stop is a future concern). Returns { ok: true }.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Hub HTTP API <- Next.js | Internal Docker network. Untrusted body content from Next.js (could be user-supplied). |
| Hub <- Agent WS | Agent responses parsed as JSON. Could be malformed. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07 | Tampering | HTTP API body injection | mitigate | Zod validation on all POST request bodies in routes.ts. Reject with 400 on invalid input. |
| T-03-08 | Tampering | Invalid agent messages | mitigate | OpenClawInbound Zod schema validation in message-router.ts. Drop and log invalid frames. |
| T-03-09 | Denial of Service | Unbounded event writes | accept | Events are append-only and bounded by agent message rate. No user-controlled amplification. DB will grow linearly with usage. |
| T-03-10 | Information Disclosure | Redis channel sniffing | accept | Redis is on internal Docker network. No external access. Channel names include tenantId for namespacing but not for security. |
</threat_model>
