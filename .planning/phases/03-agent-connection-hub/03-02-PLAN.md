<task type="auto">
  <name>Task 1: Install OpenClaw WebSocket plugin on all agents via SSH</name>
  <files></files>
  <read_first>
    .planning/phases/03-agent-connection-hub/03-RESEARCH.md (Plugin Installation section, OpenClaw WebSocket Plugin Protocol section)
  </read_first>
  <action>
SSH into the Agents VPS at 46.225.56.122 (user: root).

**1. Verify OpenClaw installation and agent processes:**
```bash
ssh root@46.225.56.122
openclaw --version   # Should be v2026.3.2
openclaw agents list # List running agents (Mo, Sam, Herman)
```

**2. Install the WebSocket channel plugin globally:**
```bash
openclaw plugins install @taichi-labs/openclaw-websocket
```

**3. Configure each agent's websocket channel.** Find each agent's config directory. For each agent, add or update the channels.websocket block in their config:

**Mo (port 18789):**
```json
{
  "channels": {
    "websocket": {
      "enabled": true,
      "port": 18789,
      "host": "0.0.0.0",
      "path": "/ws",
      "auth": {
        "enabled": false
      }
    }
  }
}
```

**Sam (port 18790):**
Same config but port 18790.

**Herman (port 19000):**
Same config but port 19000.

**4. Restart each agent** to pick up the new channel config:
```bash
openclaw agents restart mo
openclaw agents restart sam
openclaw agents restart herman
```

**5. Verify each agent is listening on its WebSocket port:**
```bash
curl -s -o /dev/null -w "%{http_code}" http://localhost:18789/ws  # Should get 426 (Upgrade Required) or similar
curl -s -o /dev/null -w "%{http_code}" http://localhost:18790/ws
curl -s -o /dev/null -w "%{http_code}" http://localhost:19000/ws
```

If curl gets a connection or a 426 response, the WebSocket endpoint is listening. A connection refused means the plugin is not active.

**IMPORTANT:** Do not modify any other agent configuration. Only add the websocket channel. Existing WhatsApp channel config must remain untouched.

**If the plugin is not available** (openclaw plugins install fails): Check npm registry for @taichi-labs/openclaw-websocket. If it truly doesn't exist, this is a blocker -- report back and we may need to build a minimal channel plugin.
  </action>
