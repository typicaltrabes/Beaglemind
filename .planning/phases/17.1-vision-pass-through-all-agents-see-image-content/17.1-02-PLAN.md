## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-1-02-01 | Tampering / Prompt-Injection | description → agent prompt block | mitigate | The `--- USER ATTACHMENTS ---` and `--- END ATTACHMENTS ---` sentinels make the user-controlled boundary explicit to agents. The `Description:` prefix further telegraphs that this content was machine-generated from user input. The system-level groupContext in routes.ts:179 already instructs agents that user-provided content is untrusted. Same disposition as T-17-03-04 (extracted text from PDFs) — accepted user-trust level. |
| T-17-1-02-02 | Information Disclosure | description leak across runs | mitigate | The SELECT WHERE clause already pins (artifact.runId === runId) AND (agentId === 'user'); 17.1-02 adds no new query — same strict-404 ownership path applies. Description is per-artifact-row; cross-run reuse fails. |
| T-17-1-02-03 | Repudiation | description was wrong / agents misled | accept | The artifacts row is the audit trail (createdAt + description text). UAT-17-1-01 verifies the description reaches agents. Description quality issues (e.g., Haiku misreading the image) are a model-quality concern, not a security concern; planning notes "iterate prompt during UAT". |
