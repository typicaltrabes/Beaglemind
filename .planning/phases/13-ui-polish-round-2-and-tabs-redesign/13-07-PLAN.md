    ```bash

    ```

    ```
    Added shared.users.preferences
    Added tenant_<id1>.runs.title
    Added tenant_<id2>.runs.title
    ...
    Phase 13 migration complete (N tenants).
    ```
    The migration is idempotent — re-running is safe. If the script errors, capture the message and STOP.

    F) **Build and recreate the containers.**
    ```bash
    ssh beaglehq "cd /tmp/beagle-build && docker compose build console-web console-worker"

    ssh beaglehq "cd /tmp/beagle-build && docker compose ps"
    ```

    Run each scenario at `https://console.beaglemind.ai`. Note pass/fail and any defects.

    **UAT-13-01 — horizontal scroll**
    1. Open browser at viewport width ≥1024px (1280×800 recommended).
    2. Navigate to any run page (`/projects/.../runs/...`).
    3. Confirm: `document.documentElement.scrollWidth === document.documentElement.clientWidth` (DevTools Console). No horizontal scrollbar visible.
    4. Switch tabs to Timeline / Boardroom / Canvas. Re-confirm at each tab.

    **UAT-13-02 — run title**
    1. Create a new run with a long prompt (>120 chars, e.g. "Look at TSLA's Q4 earnings, the analyst expectations going into them, and what the most recent insider activity tells us about management's confidence").
    2. Within ~10s, refresh the run page. Confirm the h1 shows a 6–8 word AI title (NOT the truncated prompt).
    3. Open Run History. Confirm the column header reads `Title` (not `Prompt`). Confirm the new run's Title cell shows the AI title.
    4. (Negative path): Disable LiteLLM (e.g. comment out the env var and recreate the container, OR test in dev). Create a new run. Confirm the h1 falls back to truncated prompt with no error visible to the user.

    **UAT-13-03 — improve prompt button**
    1. Open a run page. Type a draft prompt in the composer.
    2. Click the Sparkles icon (left of Send). Popover opens. Spinner shows briefly, then the rewrite text appears.
    3. Click "Use rewrite" — popover closes, composer text is replaced with the rewrite, focus returns to the textarea.
    4. Re-open with a new draft. Click "Edit and keep" — composer is replaced AND popover closes; you can keep editing.
    5. Re-open. Click "Cancel" — popover closes, composer text untouched.
    6. Empty composer → Sparkles button is disabled (visible greyed-out cursor on hover).
    7. (Negative path): If LiteLLM is down, the popover opens, spinner is shown, then the error text appears in red; Use/Edit buttons are disabled, Cancel still works.

    **UAT-13-04 — avatar edge padding**
    1. Open a run page with at least one agent message in Writers' Room.
    2. Inspect the leftmost edge of an avatar. Confirm visually that the avatar starts ~16px inside the message-column panel border (not flush against it).

    **UAT-13-05 — run id chip**
    1. Open a run page. Confirm the header shows `#` followed by 8 monospace characters (e.g. `#275423d3`), NOT the full UUID.
    2. Hover the chip. Browser tooltip shows the full UUID.
    3. Click the chip. Visible text changes to "Run ID copied" for ~1.5s, then reverts to `#xxxxxxxx`.
    4. Paste from clipboard. Confirm the FULL UUID is in the clipboard (not the slug).

    **UAT-13-06 — settings page**
    1. Click the Settings icon in the header (next to Logout). Land on `/settings`.
    2. Change theme to "Light". Within ~500ms, the page background flips to light. Hard reload — light theme persists.
    3. Change theme to "Auto". Use OS-level color preference toggle (or DevTools "emulate prefers-color-scheme: light"). Page should follow.
    4. Change theme back to "Dark".
    5. Change Default tab to "Timeline". Open a fresh run with no `?view=` in the URL — Timeline is active. URL stays clean (no `?view=`).
    6. Change Default verbosity to "Quiet". Open a fresh run, switch to Studio mode. The verbosity slider starts at the leftmost position.
    7. Change Browser notifications to "On". Open the app on a fresh device or new private window — push permission prompt appears (if push is supported).

    **UAT-13-07 — tab differentiation**
    1. Open a run with several events (or use an in-flight run).
    2. **Timeline tab:** confirm play/pause/scrub bar at the bottom. Click Play — the playhead advances one event every ~200ms (default 2×). Toggle to 1× — tempo halves. Toggle to 4× — tempo doubles. Pause auto-stops on the last event. Scrub the slider — playback pauses; detail pane below shows the event at the playhead.
    3. **Boardroom tab:** desktop view shows a grid with rows = scenes (or one "Run" row if no scenes) and cols = agents. Empty cells show a faint horizontal rule (not whitespace). Resize browser to <768px — accordion view appears.
    4. **Canvas tab:** for a run with no artifacts, the empty-state card appears with title "Canvas: deliverables view", body text, and the run prompt at the bottom in quotes. For a run with artifacts (if any are available), the preview pane + comment rail behavior is unchanged.

    **Type "approved" if every scenario passes.** Otherwise, list the specific failures by UAT-id (e.g. "UAT-13-03 fails: Use rewrite button doesn't focus textarea after closing").
