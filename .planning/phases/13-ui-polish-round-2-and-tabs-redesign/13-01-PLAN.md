<objective>
Wave-1 foundation plan. Adds two additive schema columns that the rest of Phase 13 consumes: `runs.title` (nullable VARCHAR(80)) and `users.preferences` (JSONB NOT NULL DEFAULT '{}'::jsonb). Surfaces both through the existing run-detail and run-history HTTP endpoints + their TanStack Query hooks so downstream plans (13-03 title summarization, 13-05 settings page) can read/write these fields without schema work.

Purpose: Per CONTEXT.md `<decisions>` Item 2 ("add `title VARCHAR(80)` column to `runs` table via migration") and Item 6 ("add `preferences JSONB NOT NULL DEFAULT '{}'::jsonb` to `users` table"). Phase 4 D-02 confirmed `runs.title` was historically removed from the schema in Phase 4 — `packages/db/src/schema/tenant.ts` line 19-29 confirms today's `runs` table has no `title` column. Both columns are additive and nullable/defaulted, so production tenants can be migrated without downtime.

Output:
- Drizzle source-of-truth schema files updated to declare the new columns.
- `apps/web/app/api/runs/[id]/route.ts` projects the new title field into its select.
- `apps/web/app/api/runs/history/route.ts` projects the new title field into its select.
- `apps/web/lib/hooks/use-run.ts` Run interface gains `title: string | null`.
- `apps/web/lib/hooks/use-run-history.ts` RunHistoryItem interface gains `title: string | null`.

Iterates `shared.tenants` and applies `ALTER TABLE tenant_<id>.runs ADD COLUMN IF NOT EXISTS title varchar(80)`, plus `ALTER TABLE shared.users ADD COLUMN IF NOT EXISTS preferences jsonb NOT NULL DEFAULT '{}'::jsonb`.     A) Create `packages/db/src/scripts/migrate-13.ts` with EXACTLY this content. The script mirrors the iteration pattern in `packages/db/src/migrate.ts` (lines 28-43) and uses raw SQL for idempotent column adds:

    ```typescript
    import { sql } from 'drizzle-orm';
    import { drizzle } from 'drizzle-orm/postgres-js';
    import postgres from 'postgres';

    const connectionString =
      process.env.DATABASE_URL ??
      'postgresql://postgres:postgres@localhost:5432/beagle_console';

    /**
     * Phase 13 schema migration (idempotent).
     *
     * Adds two additive columns:
     *   1. tenant_<id>.runs.title           VARCHAR(80) NULL
     *   2. shared.users.preferences         JSONB NOT NULL DEFAULT '{}'::jsonb
     *
     * Both use ADD COLUMN IF NOT EXISTS, so re-running is a no-op.
     * Per CONTEXT.md: "Both additive, nullable / defaulted, so no downtime risk."
     */
    export async function migratePhase13() {
      const client = postgres(connectionString, { max: 1 });
      const db = drizzle(client);

      try {
        // 1. Add preferences column to shared.users (single statement).
        await db.execute(sql`
          ALTER TABLE shared.users
          ADD COLUMN IF NOT EXISTS preferences jsonb NOT NULL DEFAULT '{}'::jsonb
        `);
        console.log('Added shared.users.preferences');

        // 2. Iterate every tenant schema and add runs.title.
        const tenantRows = (await db.execute(
          sql`SELECT id FROM shared.tenants`,
        )) as unknown as { id: string }[];

        for (const row of tenantRows) {
          // Sanitize tenant id the same way createTenantSchema does (hyphen -> underscore).
          const schemaName = `tenant_${row.id.replace(/-/g, '_')}`;
          await db.execute(sql`
            ALTER TABLE ${sql.identifier(schemaName)}.runs
            ADD COLUMN IF NOT EXISTS title varchar(80)
          `);
          console.log(`Added ${schemaName}.runs.title`);
        }

        console.log(`Phase 13 migration complete (${tenantRows.length} tenants).`);
      } finally {
        await client.end();
      }
    }

    // Allow direct invocation: pnpm --filter @beagle-console/db exec tsx src/scripts/migrate-13.ts
    if (import.meta.url.endsWith(process.argv[1] ?? '')) {
      migratePhase13().catch((err) => {
        console.error('Phase 13 migration failed:', err);
        process.exit(1);
      });
    }
    ```
