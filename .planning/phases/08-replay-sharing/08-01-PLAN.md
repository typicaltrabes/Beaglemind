  <action>
1. In `packages/db/src/schema/tenant.ts`, add two new tables inside `createTenantSchema`:

   **shareLinks** table (per D-02):
   - id: uuid PK defaultRandom
   - runId: uuid NOT NULL, references runs.id
   - token: text NOT NULL (unique index)
   - createdBy: uuid NOT NULL
   - expiresAt: timestamp with timezone NOT NULL
   - revokedAt: timestamp with timezone (nullable)
   - createdAt: timestamp with timezone defaultNow NOT NULL

   **replayViews** table (per D-10):
   - id: integer PK generatedAlwaysAsIdentity
   - shareLinkId: uuid NOT NULL, references shareLinks.id
   - viewerIp: text NOT NULL
   - userAgent: text
   - viewedAt: timestamp with timezone defaultNow NOT NULL

   Add unique index on shareLinks.token. Add both tables to the return object.

2. Create `apps/web/app/api/share-links/route.ts`:
   - POST handler: requireTenantContext(), accept JSON body { runId: string, expiresInDays?: number }
   - Validate the run exists and belongs to tenant (query runs table by runId)
   - Generate token: `crypto.randomBytes(32).toString('hex')` (import from node:crypto)
   - Default expiresInDays to 30 (per D-03). Calculate expiresAt = now + expiresInDays days
   - Insert into shareLinks table
   - Return 201 with { id, token, url: `${process.env.NEXT_PUBLIC_APP_URL}/replay/${token}`, expiresAt }
   - GET handler: requireTenantContext(), query all share links for the tenant (join with runs to scope). Return array of share links with run info.

3. Create `apps/web/app/api/share-links/[id]/route.ts`:
   - DELETE handler: requireTenantContext(), set revokedAt = now() on the share link by id. Return 200.
   - Validate the share link belongs to the tenant before revoking.

Use `export const runtime = 'nodejs'` and `export const dynamic = 'force-dynamic'` on both route files, matching existing API patterns.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Authenticated user -> share-links API | Only authenticated tenant members can create/revoke share links |
| Share link token -> replay content | Token grants read-only access to filtered content (Plan 02) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Spoofing | POST /api/share-links | mitigate | requireTenantContext() validates session before link creation |
| T-08-02 | Information Disclosure | share link token | mitigate | 32-byte random hex (256 bits entropy) makes brute-force infeasible |
| T-08-03 | Elevation of Privilege | DELETE /api/share-links/[id] | mitigate | Validate share link belongs to requesting tenant before revocation |
| T-08-04 | Tampering | runId in POST body | mitigate | Verify run exists in tenant schema before creating share link |
</threat_model>
