<threat_model>

| Threat | Disposition | Mitigation |
|--------|-------------|-----------|
| Migration runs against schema with existing in-flight runs (currentRound NULL by default — fine) | mitigate | DEFAULT 3/7/1500 fills existing rows; NULL is the expected value for currentRound on completed/cancelled runs. |
| Container rebuild order: hub starts before worker, schedules a job, worker hasn't booted → job waits in Redis (acceptable) | accept | BullMQ delayed jobs are durable; the worker picks them up on boot. Worst case: a 5-second delay on first idle-timeout fires. |
