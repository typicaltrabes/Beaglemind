<objective>
Implement mid-conversation message queueing. When the user posts a message while a round is in flight, the message must NOT interrupt the current agent's call but must be picked up by the next round's GROUP DISCUSSION block. Multiple queued messages between rounds concatenate.

Per CONTEXT.md decisions:
- Detect round-in-progress via `runs.current_round IS NOT NULL` (set by Plan 19-01's runRoundTable at round-start, cleared by Plan 19-02's idle-timeout watcher and by the round-table when it exits the rounds loop — see Task 1).
- Persist immediately so the message shows in transcript with normal ordering, but mark `metadata.queuedForNextRound = true`.
- At round-start, query for queued messages, prepend, clear flag, continue.
- If multiple queued messages between rounds: concatenate in send-order under a single `User:` prefix.
- If run is in idle-timeout state (rounds finished, watcher pending): treat user message as trigger for another N rounds — bypass the queue.

Wave 4 because: depends on Plan 19-01 (current_round column + multi-round loop) and Plan 19-04 (the file ownership of routes.ts is sequential — 19-01 owns the multi-round refactor, 19-04 owns the continueOnly addition, 19-05 owns the queue read).

Output:
- Messages route branches on current_round state
- runRoundTable consumes queued messages at the START of each round (after round 1, before agent loop)
- Vitest covers: in-flight queueing, multi-message concat, queue-clear after consumption, idle-state bypass
</objective>

1. Add a helper to consume queued messages. In `apps/agent-hub/src/http/routes.ts`, after `loadRunConfig`:
   ```ts
   /**
    * Phase 19-05: pull any user messages that were posted during the prior
    * round and marked `metadata.queuedForNextRound = true`. Returns the
    * concatenated text under a single `User:` prefix, or empty string if
    * nothing queued. Atomically clears the flag on consumed rows.
    */
   async function consumeQueuedMessages(
     eventStore: EventStore,
     tenantId: string,
     runId: string,
   ): Promise<string> {
     const events = await eventStore.list(tenantId, runId, { limit: 100 });
     const queued = events.filter(
       (e) =>
         e.type === 'agent_message' &&
         e.agentId === 'user' &&
         (e.metadata as Record<string, unknown> | undefined)?.queuedForNextRound === true,
     );
     if (queued.length === 0) return '';

     // Concatenate in send-order (already ASC by sequenceNumber from list()).
     const texts = queued.map((e) => (e.content as { text?: string }).text ?? '').filter(Boolean);
     const combined = texts.length === 1 ? texts[0] : texts.join('\n\n');

     // Clear the flag. Cheapest atomic way is a direct SQL update on
     // events.metadata for the matching rows, scoped to tenantId/runId.
     try {
       const { events: eventsTable } = createTenantSchema(tenantId);
       const { sql } = await import('drizzle-orm');
       await db.execute(sql`
         UPDATE ${eventsTable}
         SET metadata = jsonb_set(
           jsonb_set(metadata, '{queuedForNextRound}', 'false'::jsonb),
           '{consumedAt}', to_jsonb(now()::text)
         )
         WHERE run_id = ${runId}
           AND metadata->>'queuedForNextRound' = 'true'
       `);
     } catch (err: any) {
       log.error({ error: err.message, runId }, 'Failed to clear queuedForNextRound flag (non-blocking)');
     }

     return `User: ${combined}`;
   }
   ```

1. In `apps/web/app/api/runs/[id]/messages/route.ts`, after `requireTenantContext` and BEFORE the existing attachmentBlock-build path, add the round-in-flight branch:

   ```ts
   // Phase 19-05: detect round-in-flight state. Read currentRound from the
   // tenant runs row. NULL = idle (post-round, watcher pending) → existing
   // path triggers a fresh round-table. Non-null = round in flight →
   // queue the message for the next round's GROUP DISCUSSION.
   const runRows = await tdb
     .select({ currentRound: schema.runs.currentRound, status: schema.runs.status })
     .from(schema.runs)
     .where(eq(schema.runs.id, runId))
     .limit(1);
   const inFlight = runRows[0]?.currentRound !== null && runRows[0]?.currentRound !== undefined;

   if (inFlight) {
     // Queue path: persist the user message directly with the queue marker.
     // Compute next sequenceNumber via MAX+1 — accept a small race with the
     // hub's SequenceCounter; the unique index events_run_seq_idx is the
     // ultimate guard, and the retry below recovers from a collision.
     const { sql } = await import('drizzle-orm');
     async function nextSeq() {
       const rows = (await tdb.execute(sql`
         SELECT COALESCE(MAX(sequence_number), 0)::int + 1 AS next_seq
         FROM ${schema.events}
         WHERE run_id = ${runId}
       `)) as unknown as { next_seq: number }[];
       return rows[0]?.next_seq ?? 1;
     }

     const eventContent = attachmentIds.length > 0
       ? { text: content, attachmentIds }
       : { text: content };

     // Single retry on unique-violation, in case the hub's SequenceCounter
     // collided. After the retry, surface as 500 (fix data, not the user).
     let attempts = 0;
     while (attempts < 2) {
       try {
         await tdb.insert(schema.events).values({
           runId,
           sequenceNumber: await nextSeq(),
           type: 'agent_message',
           agentId: 'user',
           content: eventContent,
           metadata: { queuedForNextRound: true },
         });
         break;
       } catch (err: any) {
         attempts++;
         if (attempts >= 2) throw err;
       }
     }

     return NextResponse.json({ ok: true, queued: true });
   }

   // Idle path: fall through to existing logic — attachmentBlock build,
   // runs.status='executing' update, hubClient.startRun call.
   ```
