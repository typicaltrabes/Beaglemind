<objective>
Close UAT-12-01 — Run History page renders in light theme on first paint while every other dashboard view is dark. Make the dark palette unconditional in `apps/web/app/globals.css` by copying every CSS custom property from the `.dark` block into `:root`. This eliminates the hydration window during which the light palette can be visible and removes any need for theme-detection logic.

Purpose: The product has no light theme by design (per CONTEXT.md `<decisions>` §Defect 1 — "Out of scope: Adding a user-facing light/dark toggle"). The simplest fix is to make light theme impossible to render. Setting `:root` to the same values as `.dark` makes `<html className="dark">` (already in `app/layout.tsx`) idempotent rather than load-bearing.

Output:
- `:root` in `apps/web/app/globals.css` contains the exact same oklch values as `.dark` for every shared variable (`--background`, `--foreground`, `--card`, `--card-foreground`, `--popover`, `--popover-foreground`, `--primary`, `--primary-foreground`, `--secondary`, `--secondary-foreground`, `--muted`, `--muted-foreground`, `--accent`, `--accent-foreground`, `--destructive`, `--border`, `--input`, `--ring`, `--chart-1` through `--chart-5`, `--sidebar`, `--sidebar-foreground`, `--sidebar-primary`, `--sidebar-primary-foreground`, `--sidebar-accent`, `--sidebar-accent-foreground`, `--sidebar-border`, `--sidebar-ring`)
- `--radius: 0.625rem` stays in `:root` (it has no light/dark variant)
- The `.dark` block is preserved as-is (defensive — anything that explicitly targets `.dark *` keeps working)
</objective>

Notes:
- `<html>` already has `dark` class hardcoded — this fix does NOT remove it. We're making light theme impossible at the CSS level so the class becomes redundant rather than required.
