**1. Run Store: apps/web/lib/stores/run-store.ts** (per D-18, Claude's discretion on structure)

```typescript
'use client';
import { create } from 'zustand';
```

Note: Do NOT use immer middleware with Map (research assumption A3 -- may not work). Use plain object `Record<number, HubEventEnvelope>` instead of Map for the events store.

Define interface `RunState`:
- `runId: string | null`
- `events: Record<number, HubEventEnvelope>` -- keyed by sequenceNumber
- `eventOrder: number[]` -- sorted sequence numbers for rendering
- `lastSequence: number` -- for dedup
- `status: RunStatus` -- current run status (import from state-machine.ts)

Define derived getters (computed on each appendEvent):
- `plan: HubEventEnvelope | null` -- last event where type === 'plan_proposal'
- `unansweredQuestions: HubEventEnvelope[]` -- events where type === 'question' and content.answer is falsy
- `artifacts: HubEventEnvelope[]` -- events where type === 'artifact'
- `messages: HubEventEnvelope[]` -- all events for rendering (eventOrder mapped to events)

Define actions:
- `initRun(runId: string)`: reset all state, set runId
- `appendEvent(event: HubEventEnvelope)`: skip if sequenceNumber <= lastSequence (dedup). Add to events record, push to eventOrder, update lastSequence. Update derived state: if type==='state_transition' update status from content.to. If type==='plan_proposal' set plan. Recompute unansweredQuestions and artifacts arrays.
- `appendEvents(events: HubEventEnvelope[])`: batch version for replay, same logic
- `updateQuestion(questionId: string, answer: string)`: find question event in events, update its content.answer -- used for optimistic UI when user answers
- `reset()`: clear all state

Export `useRunStore`.

Export `useRunStream(runId: string | null)`:
- Get `appendEvent` and `initRun` from `useRunStore`
- `useEffect` keyed on `[runId]`:
  - If !runId, return
  - Call `initRun(runId)` to reset store
  - Create `new EventSource(/api/runs/${runId}/stream)`
  - `es.onmessage = (event) => { appendEvent(JSON.parse(event.data)); }`
  - `es.onerror` -- EventSource auto-reconnects, no manual logic needed. Log error for debugging.
  - Cleanup: `es.close()`
- Return nothing (hook is a side effect)

**4. QueryProvider: apps/web/components/providers/query-provider.tsx**

```typescript
'use client';
import { QueryClient, QueryClientProvider } from '@tanstack/react-query';
import { useState } from 'react';

export function QueryProvider({ children }: { children: React.ReactNode }) {
  const [queryClient] = useState(() => new QueryClient({
    defaultOptions: {
      queries: {
        staleTime: 30_000, // 30 seconds
        refetchOnWindowFocus: false,
      },
    },
  }));

  return (
    <QueryClientProvider client={queryClient}>
      {children}
    </QueryClientProvider>
  );
}
```
