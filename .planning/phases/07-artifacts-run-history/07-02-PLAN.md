    1. Create `apps/web/app/api/runs/history/route.ts`:
       - `export const runtime = 'nodejs'`
       - GET handler with `requireTenantContext()` + `getTenantDb(tenantId)`
       - Accept query params: `status` (optional, comma-separated), `search` (optional string), `limit` (default 50), `offset` (default 0)
       - Query: SELECT runs.*, projects.name as projectName, with LEFT JOIN on projects (runs.projectId = projects.id)
       - Also compute per-run: artifact count via subquery `SELECT count(*) FROM artifacts WHERE runId = runs.id`
       - Also compute per-run cost: `SELECT COALESCE(SUM((metadata->>'costUsd')::numeric), 0) FROM events WHERE runId = runs.id AND metadata->>'costUsd' IS NOT NULL` as totalCostUsd
       - Apply status filter: WHERE runs.status IN (...statuses) if provided
       - Apply search filter: WHERE (projects.name ILIKE '%search%' OR runs.prompt ILIKE '%search%') if provided
       - ORDER BY runs.createdAt DESC
       - Apply LIMIT/OFFSET
       - Return JSON: `{ runs: [...], total: count }` where each run includes: id, projectId, projectName, prompt, status, kind, totalCostUsd, artifactCount, createdAt, updatedAt
       - Use Drizzle's sql template tag for the subqueries (import `sql` from drizzle-orm)
       - Compute duration: for completed/cancelled runs, calculate `updatedAt - createdAt` in seconds and include as `durationSeconds`

    2. Create `apps/web/lib/hooks/use-run-history.ts`:
       - Export `useRunHistory(params: { status?: string; search?: string; limit?: number; offset?: number })`
       - TanStack Query pattern matching use-projects.ts
       - queryKey: `['runs', 'history', params]`
       - queryFn: fetch `/api/runs/history?${searchParams}` with params serialized
       - Return typed response `{ runs: RunHistoryItem[]; total: number }`
       - Export the `RunHistoryItem` type

    3. Modify `apps/web/components/sidebar/sidebar.tsx`:

       - Use `<Link href="/runs">` with a clock/history icon (lucide-react `Clock` or `History`)
       - Style: `text-sm text-muted-foreground hover:text-foreground px-3 py-2` consistent with sidebar theme
